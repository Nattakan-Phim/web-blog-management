Project Starter Kit

เริ่ม Project ใหม่ได้ภายใน 1 คำสั่ง แทนที่จะใช้เวลาหลายวัน

❌ Before
🕐 Setup folder structure ใหม่ทุกครั้ง
🕐 ตั้งค่า ESLint, TypeScript, CI/CD ซ้ำ
🕐 Copy component จาก Project เก่า
🕐 ใช้เวลา 2–3 วันก่อนจะเริ่ม develop ได้จริง
✅ After
⚡ รัน npx create-company-app
⚡ ได้ Project ที่มี config พร้อมใช้ทันที
⚡ shared-ui ถูก wire ไว้แล้ว
⚡ เริ่ม develop feature ได้ภายในชั่วโมง
⚖️ Trade-off

ข้อดีและข้อจำกัด

ไม่มี solution ที่ดีที่สุดเพียงอย่างเดียว

แนวทาง ข้อดี ข้อจำกัด
Shared Library Bug fix ครั้งเดียว, consistency สูง ต้นทุนตั้งต้นสูง, ต้อง maintain versioning
Design Token ต่าง theme โดยไม่แตะ component code ต้องออกแบบ token structure ให้ดีตั้งแต่แรก
Starter Kit ลด setup time จากวัน → ชั่วโมง ต้อง update เมื่อ best practice เปลี่ยน
Monorepo Share code ง่าย, atomic commit CI/CD ซับซ้อนขึ้น, build time นานถ้า scale ใหญ่
🗺️ Migration Path

วิธี Adopt โดยไม่ทิ้ง Codebase เดิม

ค่อยๆ build ขึ้นมา ไม่ต้อง rewrite ทุกอย่างพร้อมกัน

1
Identify & Extract
หา Component ที่ซ้ำกันมากที่สุดใน Project ที่มีอยู่แล้ว (Button, Input, Layout) แล้วย้ายมาเป็น shared package
2
Define Design Tokens
กำหนด token structure (colors, spacing, typography) จาก Design System ที่ใช้อยู่ ให้ครอบคลุมทุก Project
3
Build Starter Kit
เอา Project ล่าสุดที่ทีมพึงพอใจที่สุดเป็น template แล้วทำ scaffolding tool ให้ generate Project ใหม่จาก template
4
Adopt ใน Project ถัดไป
ใช้ Starter Kit กับ Project ใหม่ วัดผล แล้วค่อย migrate Project เก่าทีละส่วน ไม่ต้อง big-bang rewrite
